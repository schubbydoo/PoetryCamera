1. What Are MCP Servers?

MCP (Model Context Protocol) is an open standard that connects Claude Code to external tools, databases, APIs, and services. Think of them as plugins that give Claude superpowers!

How MCP Works:
  • Servers run locally or remotely - They can be Node.js processes, HTTP services, or cloud APIs
  • Claude discovers available tools - When connected, Claude can see what each server offers
  • Tasks get delegated - Claude uses the right tool for each task automatically

Three Types of MCP Servers

Type Use When Example
HTTP Remote cloud services, APIs GitHub, Notion, Sentry
Stdio Local Node.js tools Filesystem, Docker, SQLite
SSE Deprecated - use HTTP instead N/A

2. Browser Access Setup

Enable Claude Code to control Chrome for testing web interfaces, automating workflows, and debugging.

Requirements:
  • Google Chrome installed
  • Claude in Chrome extension (v1.0.36+)
  • Claude Code CLI (v2.0.73+)
  • Paid Claude plan (Pro, Team, or Enterprise)

Step 1: Install Chrome Extension

Install the Claude in Chrome extension from the Chrome Web Store.

Step 2: Launch with Browser Access

claude --chrome

This enables browser automation for this session.

Step 3: Verify Connection

In Claude Code, type:

/chrome

This shows connection status and settings.

Step 4: Enable by Default (Optional)

To enable Chrome every time:

  1. Run /chrome in Claude Code
  2. Select "Enabled by default"
Note: This increases context usage since browser tools are always loaded. Only enable if you frequently use browser automation.

What You Can Do with Browser Access

Test Your Web Interface:

"Open localhost:8000/dashboard and test if the sensor readings display correctly. Take a screenshot of any issues."

Automate Form Filling:

"Fill out the login form with test@example.com and password123, then submit and check for validation errors."

Extract Data:

"Go to the admin panel and extract all the user statistics into a CSV file."

Debug Issues:

"Open the web interface and check the browser console for JavaScript errors when loading the page."

3. Recommended MCP Servers for Raspberry Pi

ESSENTIAL GitHub MCP Server

Manage repositories, create PRs, handle issues directly from Claude Code.

claude mcp add --transport http github https://api.githubcopilot.com/mcp/
Use Cases:
  • Create pull requests with auto-generated descriptions
  • Link code changes to GitHub issues
  • Review PRs and leave comments
  • Search repositories for code examples

RECOMMENDED Filesystem MCP Server

Access files across your entire system, not just the current project.

claude mcp add --transport stdio filesystem -- npx -y @modelcontextprotocol/server-filesystem
Use Cases:
  • Read sensor data logs from different directories
  • Access configuration files across projects
  • Manage documentation and shared resources

RECOMMENDED Docker MCP Server

Build, run, and manage Docker containers for your applications.

claude mcp add --transport stdio docker -- npx -y @modelcontextprotocol/server-docker
Use Cases:
  • Containerize web interfaces for deployment
  • Test applications in isolated environments
  • Manage multi-container setups
  • Build and push images to registries

RECOMMENDED SQLite MCP Server

Query and manage SQLite databases on your Raspberry Pi.

claude mcp add --transport stdio sqlite -- npx -y @modelcontextprotocol/server-sqlite /path/to/database.db
Use Cases:
  • Query sensor data and generate reports
  • Manage user data for web interfaces
  • Create and modify database schemas
  • Optimize queries and indexes

OPTIONAL PostgreSQL MCP Server

For larger databases with advanced features.

claude mcp add --transport stdio postgres -- npx -y @modelcontextprotocol/server-postgres \ --connection-string "postgresql://user:password@localhost:5432/mydb"

OPTIONAL Sentry MCP Server

Monitor errors and crashes in your applications.

claude mcp add --transport http sentry https://mcp.sentry.dev/mcp
Use Cases:
  • Track production errors
  • Debug crashes in web interfaces
  • Monitor application health

OPTIONAL Notion MCP Server

Sync project documentation with Notion.

claude mcp add --transport http notion https://mcp.notion.com/mcp

4. Quick Setup Commands

Complete Raspberry Pi Development Setup

# Enable browser access claude --chrome # Add GitHub for repository management claude mcp add --transport http github https://api.githubcopilot.com/mcp/ --scope project # Add filesystem for broader file access claude mcp add --transport stdio filesystem -- npx -y @modelcontextprotocol/server-filesystem --scope project # Add Docker for containerization claude mcp add --transport stdio docker -- npx -y @modelcontextprotocol/server-docker --scope project # Add SQLite for database work (adjust path) claude mcp add --transport stdio sqlite -- npx -y @modelcontextprotocol/server-sqlite /home/pi/data/app.db --scope project

Managing MCP Servers

# List all configured servers claude mcp list # Get details about a specific server claude mcp get github # Remove a server claude mcp remove github # Check server status in Claude Code /mcp

5. Configuration Files

Project Configuration (.mcp.json)

Create this file in your project root to share MCP servers with your team:

{ "mcpServers": { "github": { "type": "http", "url": "https://api.githubcopilot.com/mcp/" }, "filesystem": { "type": "stdio", "command": "npx", "args": ["-y", "@modelcontextprotocol/server-filesystem", "${CLAUDE_PROJECT_DIR}"] }, "docker": { "type": "stdio", "command": "npx", "args": ["-y", "@modelcontextprotocol/server-docker"] }, "sqlite": { "type": "stdio", "command": "npx", "args": ["-y", "@modelcontextprotocol/server-sqlite", "/home/pi/data/sensors.db"] } } }

Settings Configuration (.claude/settings.json)

Control permissions and environment variables:

{ "enableAllProjectMcpServers": true, "permissions": { "allow": [ "Bash(docker:*)", "Bash(npm run:*)", "Bash(python:*)", "Bash(git:*)", "WebFetch" ], "deny": [ "Read(./secrets/**)", "Read(./.env)", "Bash(rm -rf:*)" ] }, "env": { "DOCKER_HOST": "unix:///var/run/docker.sock", "MCP_TIMEOUT": "10000" } }

6. Practical Examples

Example 1: Test Web Interface + Create GitHub Issue

Command to Claude:

"Open localhost:8000/dashboard in the browser, test the temperature sensor display, take a screenshot, then create a GitHub issue with the screenshot if there are any problems."

What Happens:

  1. Chrome MCP opens your web interface
  2. Claude tests the UI functionality
  3. Takes screenshots of issues
  4. GitHub MCP creates an issue with findings

Example 2: Query Database + Generate Visualization

Command to Claude:

"Query the SQLite database to get all temperature readings from the past 24 hours, then create a Python script that generates a line chart and saves it as a PNG."

What Happens:

  1. SQLite MCP queries sensor data
  2. Bash runs Python matplotlib script
  3. Chart saved to filesystem

Example 3: Build Docker Container + Test + Push

Command to Claude:

"Build a Docker image for the web interface, run it on localhost:3000, use the browser to test if it loads correctly, then push the image to Docker Hub if everything works."

What Happens:

  1. Docker MCP builds the image
  2. Starts container on port 3000
  3. Chrome MCP tests the running app
  4. Docker MCP pushes to registry

7. Troubleshooting

Browser Not Connecting

# Check if extension is installed # Go to chrome://extensions # Restart Claude Code with browser enabled claude --chrome # Check connection status /chrome

MCP Server Not Found

# List all servers claude mcp list # Check if server is authenticated (for HTTP servers) /mcp # Try removing and re-adding claude mcp remove server_name claude mcp add --transport http server_name URL

NPX Package Installation Issues

# Update npm npm install -g npm@latest # Clear npm cache npm cache clean --force # Manually install the package npm install -g @modelcontextprotocol/server-filesystem

Permission Denied Errors

Add permissions to your .claude/settings.json:

{ "permissions": { "allow": [ "Bash(docker:*)", "Bash(git:*)", "WebFetch" ] } }

Quick Reference: Scope Options

  • --scope local - Just you, this project (default, in .claude/local.json)
  • --scope project - Shared with team (in .mcp.json, committed to git)
  • --scope user - All your projects (in ~/.claude/settings.json)

Best Practices

  • Start minimal - Only add servers you actually need
  • Use project scope for team settings - Store in .mcp.json
  • Keep secrets out of git - Use environment variables
  • Test locally first - Use browser to verify before deployment
  • Review large outputs - They consume context tokens
  • Check authentication - Use /mcp command regularly
