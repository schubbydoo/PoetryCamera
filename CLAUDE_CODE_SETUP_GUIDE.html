1. Overview

This guide walks you through setting up a new Raspberry Pi project so Claude Code can work on it remotely via SSH. After setup, Claude can read files, make changes, run commands, and push to GitHub - all without you needing to manually transfer files or run commands.

What You'll Accomplish:
  • Enable Claude Code to SSH into your Raspberry Pi
  • Enable automatic GitHub pushes from the Pi
  • Install Claude Code on the Pi for autonomous work
  • Set up a seamless development workflow

2. Prerequisites

What You Need:

  • Raspberry Pi - Running and connected to your network
  • SSH enabled - Already configured on your Pi
  • Network access - You can reach the Pi via SSH
  • GitHub account - For code version control
  • Local machine - Where you run Claude Code CLI
Important: Know your Raspberry Pi's IP address. Find it with: hostname -I on the Pi, or check your router's connected devices.

3. SSH Access Setup for Claude Code

This allows Claude Code running on your local machine to access your Raspberry Pi without passwords.

1Generate SSH Key on Local Machine (WSL/Linux/Mac)

On your local machine where Claude Code runs:

ssh-keygen -t ed25519 -f ~/.ssh/id_ed25519_claude -N "" -C "claude-code-wsl-$(hostname)"

This creates a passwordless SSH key specifically for Claude Code.

2Display the Public Key

View the public key that needs to be added to your Pi:

cat ~/.ssh/id_ed25519_claude.pub

Copy the entire output (starts with ssh-ed25519 AAAA...)

3Add Public Key to Raspberry Pi

SSH into your Pi manually and run:

ssh pi@YOUR_PI_IP_ADDRESS

Then on the Pi:

mkdir -p ~/.ssh chmod 700 ~/.ssh echo "PASTE_YOUR_PUBLIC_KEY_HERE" >> ~/.ssh/authorized_keys chmod 600 ~/.ssh/authorized_keys

Replace PASTE_YOUR_PUBLIC_KEY_HERE with the key from step 2.

4Test the Connection

From your local machine, test passwordless SSH:

ssh -i ~/.ssh/id_ed25519_claude pi@YOUR_PI_IP_ADDRESS 'hostname'

If it returns your Pi's hostname without asking for a password, you're all set!

4. GitHub SSH Setup

This allows the Raspberry Pi to push code to GitHub automatically.

1Generate GitHub SSH Key on Raspberry Pi

SSH into your Pi and run:

ssh-keygen -t ed25519 -f ~/.ssh/id_ed25519_github -N "" -C "PROJECT_NAME-pi-github"

Replace PROJECT_NAME with your actual project name.

2Display GitHub Public Key

On the Pi, view the public key:

cat ~/.ssh/id_ed25519_github.pub

Copy the entire output.

3Add SSH Key to GitHub

In your web browser:

  1. Go to https://github.com/settings/keys
  2. Click "New SSH key"
  3. Title: Project Name - Raspberry Pi
  4. Key type: Authentication Key
  5. Paste the public key from step 2
  6. Click "Add SSH key"

4Configure Git to Use SSH Key

On the Raspberry Pi, create/update SSH config:

cat >> ~/.ssh/config << 'EOF' Host github.com HostName github.com User git IdentityFile ~/.ssh/id_ed25519_github IdentitiesOnly yes EOF chmod 600 ~/.ssh/config

5Test GitHub Connection

On the Pi, test the connection:

ssh -T git@github.com

You should see: Hi USERNAME! You've successfully authenticated...

6Update Git Remote to SSH

In your project directory on the Pi:

cd ~/YOUR_PROJECT_NAME git remote set-url origin git@github.com:USERNAME/REPO_NAME.git git remote -v

Verify the remote now shows git@github.com:... instead of https://...

5. Installing Claude Code on Raspberry Pi

This is optional but recommended if you want Claude to work autonomously on the Pi.

1Install Node.js

On the Raspberry Pi:

sudo apt update && sudo apt install -y nodejs npm

2Install Claude Code

On the Raspberry Pi:

sudo npm install -g @anthropic-ai/claude-code

3Verify Installation

Check that Claude Code is installed:

which claude && claude --version
Note: You'll need to authenticate Claude Code with your Anthropic API key the first time you run claude on the Pi. The local Claude Code (on your machine) can also work remotely via SSH without installing on the Pi.

6. Daily Workflow

Working on Your Project with Claude Code

Option A: Remote Work from Local Machine

Claude Code on your local machine works on the Pi remotely:

  1. Start Claude Code on your local machine: claude
  2. Tell Claude about the project on your Pi
  3. Claude will SSH in and work directly on the Pi
  4. All file changes happen on the Pi
  5. Claude can commit and push to GitHub from the Pi

Option B: Direct Work on Pi

SSH into the Pi and run Claude Code there:

ssh pi@YOUR_PI_IP cd ~/YOUR_PROJECT claude

Claude works directly on the Pi, you just provide instructions.

Committing and Pushing Changes

cd ~/YOUR_PROJECT git status git add . git commit -m "Description of changes" git push origin main
With SSH keys configured: Claude Code can handle all git operations automatically, including pushing to GitHub without any password prompts!

7. Troubleshooting

SSH Connection Issues

# Test basic SSH ssh pi@YOUR_PI_IP # Test with specific key ssh -i ~/.ssh/id_ed25519_claude pi@YOUR_PI_IP # Verbose output for debugging ssh -v -i ~/.ssh/id_ed25519_claude pi@YOUR_PI_IP

GitHub Push Permission Denied

# On the Pi, test GitHub connection ssh -T git@github.com # Check git remote git remote -v # Should show git@github.com, not https://

Permission Errors

# Fix SSH directory permissions chmod 700 ~/.ssh chmod 600 ~/.ssh/authorized_keys chmod 600 ~/.ssh/config chmod 600 ~/.ssh/id_* chmod 644 ~/.ssh/*.pub

Security Best Practices:
  • Use different SSH keys for different purposes (Claude Code access, GitHub, etc.)
  • Never share your private keys (files without .pub extension)
  • Use strong passwords for your Raspberry Pi user account
  • Consider using SSH key passphrases for extra security
  • Regularly update your Raspberry Pi: sudo apt update && sudo apt upgrade

You're All Set!

With this setup complete, you can start a new project session by simply telling Claude Code about your project, and it will handle everything remotely. Claude can read code, make changes, run tests, and push to GitHub - all while the code stays on your Raspberry Pi.
